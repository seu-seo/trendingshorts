# Shortform Pulse — User Flow v3

> AS-IS (v2): 온보딩 퍼스트 → 대시보드 → 추천 → 제작  
> TO-BE (v3): 트렌드 퍼스트 → 선택 → LLM 상담 → 콘티 → 10개 루프

---

## 설계 원칙

| 원칙 | 내용 |
|------|------|
| 가치 먼저 | 첫 화면에서 즉시 트렌드 경험 — 설문·회원가입 없음 |
| 스몰 스텝 | 트렌드 선택 → 2문항 → 콘티, 3분 안에 완성 |
| 맥락 보존 | 선택한 트렌드가 LLM 상담 → 콘티까지 단절 없이 이어짐 |
| 드랍 방지 | 올릴 때마다 다음 콘티 자동 제시 — 모멘텀 유지 |

## STEP 1 — 트렌드 피드 (/)

### 화면 구성

```
┌──────────────────────────────────┐
│ [라이프] [뷰티] [요리] [정보] …  │  ← 카테고리 탭
├──────────────────────────────────┤
│ PlatformPulse                    │
│ ┌──────┐ ┌──────┐ ┌──────┐      │  ← 플랫폼별 TOP 1
│ │ YT   │ │TikTok│ │Reels │      │
│ └──────┘ └──────┘ └──────┘      │
├──────────────────────────────────┤
│ 트렌드 키워드 분석 (버블맵)       │  ← 텍스트 기반 LLM 추론
│ HOT: #직장인일상  RISING: #점심  │
├──────────────────────────────────┤
│ 트렌드 목록 (SHORTS/TIKTOK/REELS)│
│ ┌────────────────────────────┐   │
│ │ 직장인 점심 루틴 vlog      │   │
│ │ 조회수 128만 · 성장 +340% │   │
│ │ 키워드: #직장인 #점심 #vlog│   │  ← 텍스트 메타데이터 기반
│ │ 트렌드 이유: "고민→해결"  │   │  ← LLM 추론 한 줄
│ │ 제목 구조 + 직장인 공감대  │   │
│ └────────────────────────────┘   │
└──────────────────────────────────┘
```

### 데이터 한계 및 대응

> **현재 수집 가능한 데이터:** 제목, 해시태그, 조회수, 성장률, 업로드 시각  
> **수집 불가:** 영상 내용, 훅 구조, 편집 패턴, BGM

LLM이 텍스트 메타데이터만으로 트렌드 이유를 추론:

```
입력:
  제목: "직장인 점심 뭐먹지? 이거 보고 해결했어요"
  해시태그: #직장인일상 #점심추천 #vlog
  조회수: 128만 / 성장률: +340% / 3일

LLM 추론 출력:
  키워드 패턴: "문제 제기 + 해결" 제목 구조
  타깃 신호: 직장인 일상 공감대
  트렌드 이유: 이 시기 직장인 점심 콘텐츠 급상승 중
```

### 온보딩 처리 (지연 방식)

```
첫 진입
  └── localStorage 확인
        ├── 페르소나 있음 → 페르소나 카테고리로 자동 필터
        └── 없음 → "전체" 탭으로 진입 (설문 강제 없음)
              → 콘티 1개 완성 후 소프트 온보딩 3문항 유도
                 Q1. 주요 카테고리 Q2. 플랫폼 Q3. 고충
```

## STEP 2 — 트렌드 선택

### TrendActionSheet 변경사항

```
AS-IS:  "원본 보기" / "소재로 사용하기"
TO-BE:  "원본 보기" / "이 포맷으로 콘티 만들기 →"
```

- 탭 시 선택된 트렌드 → Zustand `selectedTrend` 저장
- `/chat` 라우트로 이동 (기존 `/production` 직행 제거)

## STEP 3 — LLM 고충 상담 (/chat)

### 화면 흐름

```
선택한 트렌드 배너 (SelectedTrendBanner)
    ↓
Q1: "이 영상을 보고 가장 끌린 게 뭔가요?"
    [감성/공감이 좋았어요]  [정보가 유용했어요]
    [유머/재미가 좋았어요]  [나도 이런 상황이었어요]
    ↓ (선택 즉시 Q2 노출)
Q2: "지금 영상 올리는 게 가장 힘든 이유가 뭔가요?"
    [뭘 찍어야 할지 모르겠어요]  [만들어도 반응이 없어요]
    [꾸준히 못하겠어요]          [직접 입력...]
    ↓
[나만의 콘티 만들기 →]
    ↓
POST /api/angle → 앵글 추출 → /production
```

### Q&A 매핑 → 앵글 추출 로직

| Q1 (끌린 이유) | 톤 키워드 | 앵글 방향 |
|---------------|----------|----------|
| 감성/공감 | 공감형 | 시청자 경험 소환형 오프닝 |
| 정보 유용 | 정보형 | 구체적 수치/팁 중심 |
| 유머/재미 | 유머형 | 자기비하/반전 구조 |
| 나도 이런 상황 | 현실형 | 일상 공감 상황 재현 |

| Q2 (고충) | 콘티 조정 방향 |
|-----------|--------------|
| 뭘 찍어야 할지 모름 | 트렌드 포맷 직접 복제 → 각색 |
| 만들어도 반응 없음 | 훅 구조 강화, 첫 3초 집중 |
| 꾸준히 못함 | 스마트폰 1개, 5분 촬영 가능 난이도 |
| 직접 입력 | 입력값 그대로 프롬프트 반영 |

## STEP 4 — 콘티 생성 (/production)

### 콘티 화면 구성 원칙

| 항목 | 결정 |
|------|------|
| 포맷 | 장면별 카드 (세로 스크롤) |
| 각 카드 | 장면 스케치 + 대사 + 촬영 메모 한 줄 |
| 컷 수 | 4~5컷 (15초 숏폼 기준) |
| 알고리즘 인사이트 | 콘티 최상단 1회만 — 트렌드 포인트 요약 |
| 초단위 타임라인 | 콘티 단계에서 제외 (편집 단계) |

### 장면 구성 (15초 기준)

| CUT | 파트 | 시간 | 목적 |
|-----|------|------|------|
| 1 | 훅 | 0~3s | 공감/놀람으로 이탈 방지 |
| 2 | 전환 | 3~6s | 본론 예고, 궁금증 유발 |
| 3 | 본론 | 6~12s | 핵심 내용 (3포인트 or B-roll 교차) |
| 4 | 클로징 | 12~15s | CTA, 시청자 참여 유도 |

## STEP 5 — 10개 챌린지 루프 [B안]

### 루프 흐름

```
STEP 4 콘티 완성
    ↓
"촬영했어요" 체크
    ↓
진행률 +1 (N/10)
    ↓
다음 추천 트렌드 2개 자동 제시 ──→ STEP 2로 재진입
    + "트렌드 피드로 직접 가기" ──→ STEP 1로 이동 (탈출구)
    ↓
10개 완료?
    ├── YES → 알고리즘 패턴 리포트 (POST /api/pattern-report)
    └── NO  → 루프 반복
```

### B안 선택 이유

- A안(피드로 완전 복귀): 탐색 자율성 높지만 모멘텀 끊김
- **B안(다음 트렌드 자동 추천)**: 드랍 방지에 직결, 탈출구로 자율성 보장

### 저장 방식

```typescript
interface ChallengeState {
  completed: number;             // 완료 수
  contiHistory: StoryboardItem[];
  currentConti: StoryboardItem | null;
}
// localStorage key: 'sf-pulse-challenge'
```

## 탭 구조 변경

| | AS-IS (v2) | TO-BE (v3) |
|---|---|---|
| 탭 1 | 대시보드 (/) | 트렌드 (/) |
| 탭 2 | 추천 (/recommend) | 콘티 (/production) |
| 탭 3 | 제작 (/production) | 챌린지 (/challenge) |
| 중간 화면 | — | LLM 상담 (/chat) 신규 |
