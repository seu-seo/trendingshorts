# Shortform Pulse

> 요즘 터진 영상을 해부해서, 초보도 따라 찍을 수 있는 콘티로 만들어주는 서비스.  
> **10개만 올리면 알고리즘이 보인다.**

팔로워 1만 명 미만 초기 크리에이터를 위한 숏폼 콘텐츠 기획 도구.

Live Demo: https://raw.githack.com/seu-seo/trendingshorts/main/demo/v5/index.html

---

## What

YouTube Shorts · TikTok · Instagram Reels의 실시간 트렌드를 한 곳에서 비교하고, 내 유형에 맞는 트렌드를 선택해, LLM 고충 상담을 통해 나만의 콘텐츠 앵글을 뽑아내고, 광고 콘티 스타일의 장면 기획안까지 받아볼 수 있는 도구입니다.

기존 도구가 *"무엇이 인기인가"* 까지만 보여주는 것과 달리,  
본 제품은 *"그래서 나는 오늘 뭘 찍어야 하는가"* 까지 워크플로우를 확장합니다.

---

## Why — User Pain Point

**"콘텐츠를 만들었는데 아무 반응이 없다. 도대체 터지는 영상은 뭐가 다른 거야?"**

소비자 모드에서 크리에이터로 전환하는 시점에 구조적 정보 격차가 발생합니다.

```
소비자 모드                        크리에이터 모드
────────────                     ─────────────────
개인 선호 콘텐츠 노출               대중 선호 콘텐츠 필요
알고리즘이 취향 학습                알고리즘 진입 기준 불투명
내 피드 = 내 버블                  내 피드 ≠ 트렌드 데이터
```

개인화 알고리즘은 본인의 소비 패턴만 반영할 뿐, 대중의 집계된 행동을 보여주지 않습니다.  
자체 데이터 인프라를 갖춘 기성 크리에이터(팔로워 10만+)는 자체 해결 가능하지만,  
**팔로워 1만 명 미만의 초기 크리에이터는 이 격차를 해소할 수단이 없습니다.**

초기 크리에이터가 드랍하는 주요 순간 4가지:

| 순간 | 상황 | Shortform Pulse의 대응 |
|------|------|----------------------|
| 포기 1 | 시작이 너무 복잡해 | 트렌드 피드가 첫 화면 — 설문 없이 즉시 가치 경험 |
| 포기 2 | 저 사람들처럼 못 해 | "왜 터졌나" 추론 → "나도 이거 할 수 있겠다" |
| 포기 3 | 만들었는데 반응 없어 | 텍스트 기반 트렌드 이유 + "10개는 올려야 데이터" |
| 포기 4 | 다음에 뭐 올리지? | 올리면 다음 콘티 자동 추천 → 모멘텀 유지 |

## How — v3 User Flow

```
트렌드 피드 (/)
    │  로그인·설문 없이 즉시 진입
    │  플랫폼별 TOP + 텍스트 기반 트렌드 키워드 분석
    ↓
트렌드 선택 (TrendActionSheet)
    │  "이 포맷으로 콘티 만들기" CTA
    ↓
LLM 고충 상담 (/chat)
    │  Q1. 이 영상에서 끌린 이유?
    │  Q2. 지금 올리기 가장 힘든 이유?
    │  → 콘텐츠 앵글 + 톤 키워드 + 훅 문장 추출
    ↓
콘티 생성 (/production)
    │  트렌드 포맷 × 나의 앵글 → 4컷 장면 기획안
    │  장면 스케치 + 대사 구성
    ↓
10개 챌린지 루프 (/challenge)
       올림 체크 → 다음 추천 트렌드 2개 자동 제시 [B안]
       직접 피드 탐색 옵션도 유지
       10개 완료 → 알고리즘 패턴 리포트
```

## Competitive Differentiation

### vs. VidIQ · TubeBuddy (트렌드 모니터링 서비스)

| | VidIQ / TubeBuddy | Shortform Pulse |
|---|---|---|
| 핵심 기능 | 트렌드 데이터 조회 | 트렌드 → 나만의 콘티까지 |
| 시작점 | 채널 데이터 분석 | 아이디어 단계 |
| 개인화 | 채널 성과 기반 | 내 고충 × 트렌드 매칭 |
| 출력물 | 키워드·통계 리포트 | 오늘 찍을 수 있는 콘티 |
| 타깃 | 성장 중인 크리에이터 | 초기 크리에이터 (1만↓) |

### vs. 캡컷 · 릴스 템플릿 (편집 툴)

| | 캡컷 / 릴스 템플릿 | Shortform Pulse |
|---|---|---|
| 시작점 | 영상 파일이 있어야 시작 | 트렌드 발견 → 아이디어 단계 |
| 핵심 가치 | 빠른 편집 완성 | 뭘 찍을지 알게 됨 |
| 개인화 | 없음 (동일 템플릿) | 내 유형 × 내 고충 × 트렌드 |
| 알고리즘 | 없음 | 텍스트 기반 패턴 추론 |
| 지속성 | 1회성 | 10개 루프 → 패턴 학습 |

### USP 한 줄 정의

> **"요즘 터진 영상의 키워드 패턴을 해부해서, 초보도 따라 찍을 수 있는 콘티로 만들어주는 서비스. 10개만 올리면 알고리즘이 보인다."**

## Stack

| 레이어 | 기술 |
|--------|------|
| Frontend / Backend | Next.js 14 (App Router + API Routes) + TypeScript |
| 상태 관리 | Zustand + localStorage |
| 외부 데이터 | YouTube Data API v3, Apify (TikTok · Instagram 스냅샷) |
| AI | **Gemini 2.5 Flash** (`@ai-sdk/google`) — 앵글 추출, 콘티 생성, 인사이트 |
| 호스팅 | Vercel (Seoul 리전 icn1) |

> v3에서도 Gemini 2.5 Flash 유지 — 한국어 콘텐츠 생성 품질 및 비용 효율성
