## 1. 한 줄 요약

트렌드 데이터를 컨텐츠 마케팅용 영상 대본 초안(Hook + 본문 + CTA)으로 변환하는, 파이프라인의 **마지막 마디** 피처.

---

## 2. 왜 이 피처가 필요한가

### 2.1. 우리 매니페스토와의 연결

매니페스토 핵심 가치 #2 "감이 아닌 데이터로 결정"의 가장 마지막 단계 — **데이터가 의사결정을 거쳐 실제 산출물로 떨어지는 지점**. 트렌드를 발견하고 분석하는 것까지는 갔지만, 그것이 사용자의 실제 컨텐츠 제작으로 이어지지 않으면 매니페스토는 표어로 남는다.

### 2.2. 팀 파이프라인에서의 위치

```
발견 (규동, 백엔드)
    ↓
분석 (승연, 카테고리별 인게이지먼트 비교)
    ↓
개인화 (지은, 페르소나)
    ↓
실행 (본 피처, Output Layer)  ← End of pipeline
```

이 피처가 없으면 팀 작업이 "대시보드"로 끝난다. 이 피처가 있어야 **사용자가 실제로 컨텐츠를 만들기 위한 도구**가 된다.

### 2.3. 사용자 페인 포인트

- **크리에이터**: 트렌드를 보긴 했는데, "그래서 내가 뭘 찍어야 하지?"에서 막힌다
- **마케터**: 트렌드 데이터와 자기 브랜드 사이의 변환 작업이 매번 수작업

## 3. V0 (HW6 시점, 2026-04-30 ship)

### 3.1. Scope

- ✅ 단일 트렌딩 영상 → 3가지 톤 변주 대본 생성
- ✅ Hook (첫 3초 멘트) + 본문 흐름 + CTA 구조
- ✅ 정보형 / 스토리형 / 후킹형 톤
- ✅ Anthropic Claude API 직접 호출 (sessionStorage로 키 보관)
- ✅ 한국어 UI + 5가지 에러 케이스 처리
- ❌ 백엔드 미연결 (샘플 데이터 하드코딩)
- ❌ 다중 카테고리 (오럴케어 1개)
- ❌ 페르소나 통합

### 3.2. 결과물

- **코드**: https://github.com/kaist9558/kaist9558.github.io/tree/feature/hw6-output-layer/hw6
- **라이브 데모**: https://kaist9558.github.io/hw6/
- **AI 코드 리뷰**: `hw6/REVIEW_v0.md`

### 3.3. V0에서 검증한 가설

- ✅ Anthropic API에 트렌드 메타데이터를 컨텍스트로 넘기면 카테고리 적합한 대본이 생성된다
- ✅ JSON 구조 응답을 받으면 화면에 안정적으로 분리 렌더 가능하다
- ⚠️ "3개 톤 동시 제공이 사용자에게 친절한 UX"라는 가정은 **AI 리뷰에서 반박**됨 — V1에서 재설계 필요

## 4. V1 (5/16 발표 목표)

### 4.1. AI 리뷰가 짚은 V0의 약점

| 약점 | V1 해결 방향 |
|---|---|
| 3톤 병렬 제공 → 사용자가 비교 기준 없이 감으로 1개 고름 = 매니페스토 가치 #2 위반 | 1톤 추천 + 데이터 기반 적합도 점수(0-10), 다른 톤은 토글로 |
| "키는 어디로도 전송 안 됨" UI 문구가 기술적으로 부정확 | 정직한 문구로 수정 ("브라우저에 저장, 매 요청 시 Anthropic으로 전송") |
| 화면의 인게이지먼트 +39%가 결과물에 영향 안 줌 | 인게이지먼트 수치를 시스템 프롬프트의 변수로 주입 |

### 4.2. V1 핵심 기능 추가

1. **백엔드 연결** (규동님과 협업): 샘플 데이터 → 실제 트렌딩 영상 fetch
2. **카테고리 통합** (승연님과 협업): PR #6의 8개 메인 카테고리 + 서브 카테고리 taxonomy 그대로 사용
3. **페르소나-톤 매핑** (지은님과 협업): 페르소나별로 추천 톤 자동 매칭

### 4.3. V1 외 (V2 이후)

- 영상 자동 생성
- 다국어 (영어, 일본어)
- 사용자 계정 + 히스토리
- A/B 비교 로깅

## 5. 다른 팀원 피처와의 인터페이스

### 5.1. From 규동님 (Backend)

**기대하는 input 형태** (V1에서 협의 필요):

```json
{
  "video_id": "string",
  "title": "string",
  "category": "string",
  "platform": "youtube_shorts | tiktok | instagram_reels",
  "engagement_rate": 0.082,
  "category_avg_engagement": 0.0591,
  "thumbnail_url": "string",
  "duration_sec": 30
}
```

→ 규동님 백엔드 스키마에 맞춰 조정 가능. PR 코멘트로 합의.

### 5.2. From 승연님 (Compare PR #6)

**의존하는 부분**: 카테고리 taxonomy (8 메인 × 4-5 서브)

- V1에서 K의 시스템 프롬프트가 카테고리 ID를 그대로 받아서 처리
- 카테고리별 평균 인게이지먼트도 함께 받아 프롬프트 컨텍스트에 주입

### 5.3. From 지은님 (Personalization)

**의존하는 부분**: 페르소나 ID

- 각 페르소나마다 추천 톤 1개 매핑 (예: "초보 크리에이터" → "후킹형", "전문 브랜드" → "정보형")
- V1에서 페르소나 입력값을 받아 자동으로 1순위 톤 결정
